du gestionnaire de contenu

Tout ce que vous devez savoir pour mettre à jour les textes, tarifs et photos du site. Aucune connaissance technique requise.

Pré-requis : un compte GitHub est obligatoire. Le gestionnaire de contenu utilise GitHub pour héberger les fichiers du site. Vous devez disposer d'un compte GitHub autorisé avant de pouvoir vous connecter. Si vous n'en avez pas encore, contactez le développeur du site pour obtenir un accès.

Connexion via GitHub

La connexion se fait par GitHub, la plateforme qui héberge les fichiers du site. Suivez ces étapes :

  1. Ouvrez l'adresse du gestionnaire Vous verrez la page de connexion avec un bouton bleu "Login with GitHub".
  2. Cliquez sur "Login with GitHub" Une fenêtre GitHub s'ouvre dans votre navigateur.
  3. Entrez vos identifiants GitHub Si vous n'êtes pas encore connecté, saisissez le nom d'utilisateur et le mot de passe du compte GitHub autorisé. Demandez ces identifiants au développeur ou au responsable du site.
  4. Autorisez l'accès si demandé GitHub peut vous demander de confirmer l'accès au dépôt du site. Cliquez sur "Authorize" (Autoriser).
  5. Vous êtes connecté Après quelques secondes, l'interface principale du gestionnaire s'affiche.
Problème de connexion ?
Si la fenêtre GitHub reste ouverte sans se fermer : vérifiez que les cookies ne sont pas bloqués dans votre navigateur. Utilisez de préférence Google Chrome ou Mozilla Firefox.

Vue d'ensemble de l'interface

Une fois connecté, l'interface se divise en trois zones :

  • Menu gauche — liste des collections (types de contenu) disponibles.
  • Zone centrale — formulaire de modification du contenu sélectionné.
  • Barre haute — onglet Media pour gérer les photos, et bouton de publication.

Comment modifier un contenu

  1. Cliquez sur une collection dans le menu gauche La liste des entrées de cette collection s'affiche.
  2. Cliquez sur l'entrée Il n'y en a généralement qu'une. Le formulaire s'ouvre avec tous les champs.
  3. Modifiez les champs souhaités Le gestionnaire sauvegarde automatiquement votre travail en brouillon au fur et à mesure.
  4. Publiez la modification Cliquez sur "Publish" en haut à droite pour rendre les changements visibles sur le site. Voir section 6 pour le détail.
Sauvegarde automatique
Le gestionnaire enregistre vos modifications en brouillon (Draft) au fur et à mesure que vous tapez. Vous ne risquez pas de perdre votre travail si vous changez d'onglet accidentellement. La publication est une étape séparée et volontaire.

La médiathèque (onglet Media)

En haut de l'interface, l'onglet Media (ou une icône d'image) donne accès à la médiathèque. C'est là que vous pouvez importer de nouvelles photos pour le site. Les photos sont rangées dans le dossier Photos du Hotel et Club.

Cette section vous permet de mettre à jour les informations de contact et les prix affichés sur le site.

Accéder à ce formulaire

  1. Dans le menu gauche, cliquez sur "Coordonnées et Tarifs des Chambres"
  2. Cliquez sur l'entrée qui apparaît Le formulaire s'ouvre.

Attention — Format WhatsApp
Le numéro WhatsApp doit être en format international, sans le signe +, sans espaces et sans tirets. Un format mal saisi empêche le lien de fonctionner sur les téléphones des visiteurs. Exemple correct : 224669699999

Tarifs des chambres

Le site propose quatre types de chambres. Chaque chambre a son propre champ de prix.

Champ CMSType de chambreFormat attendu
price_standard Chambre Standard Chiffre uniquement, ex. : 150000
price_superior Chambre Supérieure Chiffre uniquement, ex. : 200000
price_deluxe Chambre Deluxe Chiffre uniquement, ex. : 300000
price_suite Suite Chiffre uniquement, ex. : 500000
Format des prix
Saisissez uniquement les chiffres, sans espace, sans virgule et sans devise. Ne tapez pas 150 000 ni 150,000. Le site ajoute automatiquement la devise et le format d'affichage.

Exemple : modifier un tarif

Vous souhaitez passer le prix de la chambre Standard de 150 000 GNF à 175 000 GNF :

  1. Trouvez le champ price_standard Il affiche actuellement 150000.
  2. Effacez la valeur et tapez 175000 Vérifiez qu'il n'y a ni espace ni symbole.
  3. Publiez la modification Suivez les étapes de la section 6.

Cette collection regroupe tous les textes visibles sur le site. Elle contient plus de 70 champs, organisés par section.

Règle absolue — Bilingue obligatoire
Le site est affiché en français et en anglais. Chaque texte existe en deux versions : un champ _fr (français) et un champ _en (anglais). Lorsque vous modifiez un texte en français, vous devez impérativement mettre à jour la version anglaise correspondante, et vice versa. Un visiteur anglophone qui voit un texte non traduit donne une mauvaise image de l'établissement.
Pas de traducteur disponible ?
Utilisez DeepL (deepl.com) ou Google Traduction pour obtenir une traduction correcte, puis relisez-la avant de publier.

Les photos du site sont stockées dans la médiathèque du gestionnaire. Elles sont organisées dans un dossier dédié : Photos du Hotel et Club.

Accéder à la médiathèque

  1. Cliquez sur l'onglet "Media" dans le menu du haut (Ou sur l'icône représentant une image, selon la version de l'interface.)
  2. Naviguez vers le dossier "Photos du Hotel et Club" Toutes les photos du site se trouvent dans ce dossier.
📷
Capture d'écran : médiathèque avec le dossier "Photos du Hotel et Club" visible

Importer une nouvelle photo

  1. Dans la médiathèque, cliquez sur "Upload" ou "Importer"
  2. Sélectionnez la photo sur votre ordinateur Vérifiez le format et la taille avant d'importer.
  3. Attendez la fin du chargement Une barre de progression s'affiche.
  4. La photo apparaît dans la médiathèque Elle est maintenant disponible pour le site.

Remplacer une photo existante

Si vous souhaitez changer une photo déjà utilisée sur le site (par exemple, la photo de la chambre Standard), deux approches sont possibles :

  • Méthode recommandée : importez la nouvelle photo avec exactement le même nom de fichier que l'ancienne. Le site utilisera automatiquement la nouvelle image sans qu'il soit nécessaire de modifier le code.
  • Méthode alternative : importez la nouvelle photo sous un nouveau nom, puis demandez au développeur de mettre à jour la référence dans le code du site.

Règles pour les photos

RègleValeur recommandée
Largeur minimale1 200 pixels
Taille maximale par fichier3 Mo
Format recommandé (photos)JPEG
Format recommandé (logos, transparence)PNG
Nom de fichierLettres minuscules, chiffres, tirets uniquement. Pas d'espace, pas d'accent, pas de caractère spécial.
Ne jamais supprimer une photo sans vérification
Ne supprimez jamais une photo de la médiathèque sans vous assurer qu'elle n'est plus utilisée sur le site. Supprimer une image utilisée provoque l'apparition d'un espace vide ou d'une icône cassée sur la page.
Photos trop petites ou trop lourdes
Une photo trop petite (moins de 1 200 px de large) apparaît floue. Une photo trop lourde (plus de 3 Mo) peut faire échouer l'importation et ralentit l'affichage du site pour les visiteurs.

Étapes pour publier

  1. Vérifiez vos modifications Relisez tous les champs modifiés. Vérifiez que la version anglaise a bien été mise à jour si vous avez modifié un texte en français.
  2. Cliquez sur le bouton "Publish" en haut à droite Le bouton peut afficher une flèche déroulante. Si c'est le cas, sélectionnez "Publish now" (Publier maintenant).
  3. Confirmez si une boîte de dialogue s'affiche Certaines versions du gestionnaire demandent une confirmation. Cliquez sur "Publish" pour valider.
  4. Attendez quelques minutes Le gestionnaire envoie les changements à GitHub, et le site se met à jour automatiquement. Le délai habituel est de 2 à 5 minutes.
  5. Vérifiez le résultat sur le site Ouvrez hotelclubdekipe.com dans un nouvel onglet. Si vous ne voyez pas les changements, actualisez la page (Ctrl + F5 ou Cmd + Shift + R).

Vous avez un doute ?
Sauvegardez en brouillon (les modifications sont déjà enregistrées automatiquement) et demandez une validation à votre responsable avant de publier. Vous ne risquez rien tant que vous n'appuyez pas sur "Publish".

Annuler une modification publiée

Si vous avez publié une erreur par inadvertance, il est possible de revenir à une version précédente. Contactez le développeur du site qui dispose de l'historique complet des versions via GitHub.

La plupart des problèmes que vous pourriez rencontrer ont une solution simple. Voici les situations les plus courantes.

Problème de connexion GitHub

  • Vérifiez que vous utilisez bien un compte GitHub autorisé à accéder au dépôt du site.
  • Si GitHub demande une vérification en deux étapes, suivez les instructions sur votre téléphone ou votre application d'authentification.
  • Essayez d'effacer les cookies de votre navigateur et de recommencer la connexion.
  • Utilisez de préférence Google Chrome ou Mozilla Firefox.

Les modifications ne s'affichent pas sur le site

  • Vérifiez que vous avez bien cliqué sur "Publish" (et non pas simplement sauvegardé en brouillon).
  • Attendez 5 minutes — la mise à jour du site peut prendre quelques minutes.
  • Actualisez la page du site avec Ctrl + F5 (Windows) ou Cmd + Shift + R (Mac).
  • Essayez depuis un autre appareil ou en navigation privée pour éliminer le cache.

Une photo n'apparaît pas ou est cassée

  • Vérifiez que le nom de fichier de la photo est identique à celui utilisé dans le code (sensible à la casse, pas d'espace).
  • Vérifiez que la photo a bien été importée dans le dossier Photos du Hotel et Club de la médiathèque.
  • Contactez le développeur si le problème persiste après vérification du nom.

Le bouton "Publish" ne répond pas

  • Rechargez la page du gestionnaire et reconnectez-vous.
  • Vérifiez votre connexion internet.
  • Contactez le développeur si le problème persiste.

Contacter le développeur

Assistance technique
Pour tout problème technique que vous ne parvenez pas à résoudre seul, contactez le développeur responsable du site. Il dispose de l'accès complet au code, aux serveurs et à l'historique des versions pour corriger tout incident rapidement.

Règles de sécurité

Important
  • Ne partagez jamais les identifiants GitHub avec des personnes non autorisées.
  • Ne supprimez pas de fichiers dans la médiathèque sans avoir vérifié qu'ils ne sont plus utilisés sur le site.
  • En cas de doute sur une modification, sauvegardez en brouillon et demandez une validation avant de publier.
  • Si vous pensez avoir fait une erreur, ne tentez pas de la corriger sans aide — contactez le développeur immédiatement.
